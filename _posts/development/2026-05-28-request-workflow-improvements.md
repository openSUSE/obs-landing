## Introduction
OBS has featured canned responses for a while now. Previously, users could only create personal canned responses under their profiles. With this latest update, we have expanded the feature so they can be shared across projects and packages.

{% include partials/_series-of-posts-about-request-redesign.md %}

The request redesign is part of [the beta program](/2018/10/04/the-beta-program/). If you want to try it out, please join the program.

## Shared Canned Responses
Until now, canned responses were strictly linked to individual users. With this new update, users can attach canned responses to projects and packages, allowing them to be shared among the reviewers of a request.

<img src="/images/posts/2026-05-06-request-workflow-improvements/shared-canned-responses.png" alt="Shared Canned Response" width="500px">

## Canned Responses Now Available for Request Reviews
Canned responses are now also available for request reviews. This means that when you are requesting a review or reviewing a request, you can use canned responses.

Currently, shared canned responses are utilized exclusively on the request page. They can be used when:
- Adding new comments to a request
- Submitting a request decision
- Requesting a review
- Reviewing a request

<img src="/images/posts/2026-05-06-request-workflow-improvements/review-canned-responses.png" alt="Request Reviews Canned Response" width="800px">

## Canned Response API Endpoints
Canned responses are fully accessible via our API. You can find the complete details in our [API documentation](https://api.opensuse.org/apidocs/index#/Canned%20Responses).
