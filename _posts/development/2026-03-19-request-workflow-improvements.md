The improvement of the Open Build Service (OBS) Request Page continues!
This update introduces several new features and bug fixes, focusing on smarter action menus and more accessible metadata.

Here’s a breakdown of what’s new in this iteration:


## Highlighting of Commented Lines

Reviewing code is now easier.
When a line in a diff gets commented, it is clearly highlighted to help you focus on the discussion.

<img src="/images/posts/2026-03-19/highlight-commented-lines.png" alt="Commented Lines Highlighting" style="width:800px" />


## Enhanced Visibility in the Accept Dropdown

The "Accept" menu has been updated to improve clarity and accessibility.
Now, all available actions are listed directly within the dropdown, including the primary action that previously appeared only on the button itself.

<img src="/images/posts/2026-03-19/dropdown-accept-button.png" alt="Enhanced Visibility in the Accept Dropdown" style="width:800px" />


## Contextual Descriptions for RPMLint Checks

No more guessing why a check failed.
We’ve integrated contextual descriptions for RPMLint results directly into the UI, providing immediate guidance on details of the issues without leaving the page.
Just hover over the little question mark next to the name of the specific lint to see the description.

<img src="/images/posts/2026-03-19/rpmlint-checks-contextual-descriptions.png" alt="Contextual Descriptions for RPMLint Checks" style="width:800px" />


## Meaningful content for Release Actions

We’ve improved how release actions are displayed.
Instead of generic or empty views, the changes tab now surfaces the most relevant information to provide better content.

<img src="/images/posts/2026-03-19/meaningful-content-release-actions.png" alt="Meaningful content for Release Actions" style="width:800px" />


## Shareable Links for Comments

Collaboration is all about communication.
You can now browse direct links to specific comments on the Conversations tab, making it easy to reference feedback in your external tool of choice.

You can find the link to each comment on its date.

<img src="/images/posts/2026-03-19/shareable-links-for-comments.png" alt="Shareable Links for Comments" style="width:800px" />


## Conclusion

These updates are now live!
Head over to your latest requests to see them in action.
Happy building!

We build OBS for you, so your feedback is always valuable.
Let us know what you think by sharing your experience.
