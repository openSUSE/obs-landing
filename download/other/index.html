Have it your way
OBS deployment methods

Every company, project and data center is different. Thats why you can deploy the OBS as packages, appliance images or source code to real hardware and different virtualization solutions. Everything is possible.

Appliances

The preferred method of deploying an Open Build Service instance are the OBS software appliances. In these appliances the OBS components are combined with the openSUSE Distribution to run optimally on a server or in a virtual machine

Installation Image

This installation image runs a basic installer that let's you choose the block device to deploy the appliance to.

You put this image on a USB device and boot from it. 

      xzcat obs-server-install.x86_64.raw.xz > /dev/sdX

where /dev/sdX is the block device corresponding to your USB device.

If you're not using Linux/*nix go checkout out Keka, 7-zip.org and the SUSE Studio Image writer

Download obs-server.x86_64.raw.install.raw.xz
VirtualBox/VMware Image

For VirtualBox and VMware there is an archive containing a configuration file (.vmx) and a disk image (.vmdk). You can load the configuration into VMware workstation/player or VirtualBox.

KVM/QEmu Image

For QEMU/KVM there is a disk image in the qcow format which you can start with qemu-kvm or any other virtualization framework which is capable of using the QCow file system.

Raw Disk Image

If you want to deploy directly to a block device of your bare-metal server or if you are using a virtualization framework that can handle raw disk images there is an file system image for you.

Packages

If you already have a Linux server running you can use our binary packages to install OBS and follow our README.Setup to set everything up yourself. Currently we maintain packages for the openSUSE and SUSE Linux Enterprise distributions.

SUSE Linux Enterprise
      zypper ar -f http://download.opensuse.org/repositories/OBS:/Server:/2.4/SLE_11_SP2/OBS:Server:2.4.repo
      zypper in -t pattern OBS_Server
openSUSE
      zypper ar -f http://download.opensuse.org/repositories/OBS:/Server:/2.4/openSUSE_12.3/OBS:Server:2.4.repo
      zypper in -t pattern OBS_Server

Source Code

You can also grab the source code from github and follow our README.Setup to set everything up yourself. 

      git clone git@github.com:openSUSE/open-build-service.git
