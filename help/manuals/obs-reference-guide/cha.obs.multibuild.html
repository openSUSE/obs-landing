11 Multibuild

11.1 Overview
11.2 How Multibuild is Defined

11.1 Overview #

Multibuild is used for sources that can build multiple packages. For that to be accomplished you must provide a file with a list of package names. For each of these listed packages a build instruction, e.g. a spec file, is needed. A set of package builds, repositories and architectures will be built for each package.

For example, with the kernel package we can build both kernel-source and kernel-obs-build packages.

11.2 How Multibuild is Defined #

Multibuild packages are defined with the _multibuild directive file in the package sources.

The _multibuild file is an xml file. For example: 

<multibuild>
  <package>gcc-package1</package>
  <package>gcc-package2</package>
</multibuild>
